Projects

Submarine Demo


This project simulates an underwater submarine expedition. I developed this project while employed at the VEMI Lab at the University of Maine. The demo was built using the Unity 3D game engine, and uses Steam's VR plug-in to render the demo into an augmented reality environment. The purpose of the demo was to showcase how real-world movement via external devices (i.e. a joystick) can be mapped to movements in an augmented reality environment. In other words, when the user moves the joystick to control the submarine, the joystick graphic that is part of the virtual submarine also moves in real-time. Also, the whale is really cool.

Foot Fault Detection System


The foot fault detection system was my computer science capstone project (basically, my "senior thesis"). I had the idea to create a foot fault detection system after having watched Serena Williams lose a semifinal US Open match largely due to a contested foot fault call back in 2009. I realized that to eliminate subjective line calls, it would be easier to have a machine make those calls as opposed to a line judge. The foot fault detection system was written in Java in the Eclipse IDE and used the OpenCV computer vision library. The GUI was created with the Java Swing graphics library. The idea behind the system was that a computer or device with the detection system software on it would be positioned on the ground looking down the baseline. The image frame would focus on the upper half of the player's body and the player's foot in relation to the baseline. The end goal is to determine whether the player's foot makes contact with the baseline before the racquet makes contact with the ball. If this case proves true, then the system outputs 'foot fault' to the screen.




EVHT 300 Web Interface


EVHT 300 SAW system. I was charged with creating a web interface that displays real-time SAW (Surface Acoustic Wave) data for my recent internship. For propriety reasons, I cannot show the web interface. However, this is the GUI of the Python desktop executable that automatically starts and stops the web interface via scripts stored on the web server. The web interface displays temperature, frequency, and power data on Google charts, and I implemented a feature that allows the user to take a screenshot of the charts, display the all the data points from the charts, and download the graph images and data points to a desktop file. This project also gave me experience working with and configuring Apache web server on Linux.
